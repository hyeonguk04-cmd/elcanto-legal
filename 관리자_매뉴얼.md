## 👥 사용자 관리

### **1. 신규 회원 승인**

#### **승인 절차**
1. 관리자 페이지 로그인
2. **"보류 중"** 탭 클릭
3. 사용자 정보 확인:
   - 이름
   - 이메일 (@elcanto.co.kr 여부)
   - 가입 일시
4. **"승인"** 버튼 클릭

#### **승인 기준**
✅ **승인 대상:**
- 엘칸토 임직원 (회사 이메일)
- 협력사 직원 (사전 협의)

❌ **거부 대상:**
- 개인 이메일 (@gmail.com 등)
- 알 수 없는 도메인
- 중복 가입

---

### **2. 사용자 상태 관리**

#### **상태 종류**
| 상태 | 설명 | 사용 가능 여부 |
|------|------|--------------|
| **승인됨 (approved)** | 정상 사용자 | ✅ 가능 |
| **보류 중 (pending)** | 승인 대기 | ❌ 불가 |
| **비활성 (disabled)** | 일시 정지 | ❌ 불가 |

#### **상태 변경**
1. 사용자 목록에서 대상 선택
2. 상태 변경 버튼 클릭:
   - **승인**: pending → approved
   - **비활성화**: approved → disabled
   - **활성화**: disabled → approved
   - **삭제**: 영구 삭제 (복구 불가)

## 🚨 장애 대응

### **일반적인 문제 해결**

#### **1. 로그인 불가**

**증상:**
- "이메일 또는 비밀번호가 올바르지 않습니다"
- "승인되지 않은 사용자입니다"

**해결:**
1. 사용자 상태 확인 (pending → approved)
2. 비밀번호 재설정 (DB에서 직접 수정)
3. 계정 재생성

---

#### **2. 검색 오류**

**증상:**
- "검색 중 오류가 발생했습니다"
- 무한 로딩
- 빈 응답

**해결:**
1. **Render 로그 확인**
   ```
   Dashboard → Logs → 최근 에러 검색
   ```

2. **Gemini API 키 확인**
   - Environment → GEMINI_API_KEY
   - Google AI Studio에서 키 상태 확인

3. **데이터베이스 연결 확인**
   - DATABASE_URL 확인
   - PostgreSQL 상태 점검

4. **서비스 재시작**
   - Render Dashboard → Manual Deploy → Deploy Latest Commit

---

#### **3. 느린 응답 속도**

**원인:**
- 데이터베이스 과부하
- Render 서버 리소스 부족
- Gemini API 응답 지연

**해결:**
1. Render 메트릭 확인 (CPU, Memory)
2. 데이터베이스 최적화 (불필요한 로그 삭제)
3. 필요시 Render 플랜 업그레이드
